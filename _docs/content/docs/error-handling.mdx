# Error Handling

Handlers return `error`. Surf renders that error to the client through a
configurable renderer. The default behavior:

- A returned `*HTTPError` becomes a JSON error envelope with its status code.
- Any other error becomes a generic 500 (internal detail is logged, never
  serialized to the client).
- A returned `surf.Abort` stops processing silently — useful when the handler
  has already written the response.
- If the handler already wrote a response before returning an error, the
  renderer is **skipped** so the response is preserved.

## HTTPError — typed responses

```go
app.Get("/users/:id", func(w http.ResponseWriter, r *http.Request) error {
    id := surf.Param(r, "id")
    user, err := store.Find(id)
    if errors.Is(err, ErrNotFound) {
        return surf.NewHTTPError(http.StatusNotFound, "user not found")
    }
    if err != nil {
        return surf.Errorf(http.StatusInternalServerError, "lookup failed", err)
        // The wrapped err is logged but never sent to the client.
    }
    return surf.JSONData(w, user)
})
```

The default renderer produces:

```json
{ "error": "user not found", "status": 404 }
```

## Abort — silent completion

When a handler has already written its response (a streaming endpoint, a
hand-rolled custom status, an SSE flush loop), return `surf.Abort` to tell the
router not to render anything further:

```go
app.Get("/events", func(w http.ResponseWriter, r *http.Request) error {
    rw := surf.GetResponseWriter(r)
    rw.Header().Set("Content-Type", "text/event-stream")
    rw.WriteHeader(http.StatusOK)
    for { /* write events */ }
    return surf.Abort
})
```

`http.ErrAbortHandler` is also honored for compatibility with the stdlib idiom.

## JSON Envelope Helpers

```go
surf.JSON(w, 200, v)                 // raw value
surf.JSONData(w, v)                  // {"data": v}
surf.JSONList(w, items, total)       // {"data": [...], "total": n}
surf.JSONError(w, 404, "not found")  // {"error": "not found", "status": 404}
```

`JSONDataStatus(w, status, v)` lets you wrap data with a non-200 status (e.g.
`201 Created`).

On the [fast path](/docs/context), the equivalent methods live on `*Context`.

## Custom Renderer

`WithErrorHandler` replaces the default renderer. Use it to integrate with
your existing error-response format:

```go
app := surf.NewApp(surf.WithErrorHandler(func(w http.ResponseWriter, r *http.Request, err error) {
    var httpErr *surf.HTTPError
    if errors.As(err, &httpErr) {
        _ = surf.JSON(w, httpErr.Code, map[string]any{
            "ok":      false,
            "code":    httpErr.Code,
            "message": httpErr.Message,
            "trace":   r.Header.Get("X-Request-ID"),
        })
        return
    }
    _ = surf.JSON(w, http.StatusInternalServerError, map[string]any{
        "ok":      false,
        "message": "internal error",
    })
}))
```

The renderer is **only invoked when the response has not been written**. A
handler that returns an error after writing keeps its response intact.

## Validation Errors

`BindAndValidate` returns a `422 *HTTPError` whose `Message` is the
validator's error message — so it flows through the renderer without extra
code:

```go
type SignupBody struct {
    Name  string `json:"name"`
    Email string `json:"email"`
}

func (b SignupBody) Validate() error {
    if b.Name == "" {
        return errors.New("name is required")
    }
    return nil
}

app.Post("/signup", func(w http.ResponseWriter, r *http.Request) error {
    var body SignupBody
    if err := surf.BindAndValidate(r, &body); err != nil {
        return err  // 400 for bad JSON, 413 over limit, 422 for validation
    }
    return surf.JSONDataStatus(w, http.StatusCreated, body)
})
```

## Custom 404 and 405

When no route matches a request, surf returns 404; when the path matches a
route registered for a different method, surf returns 405 with an `Allow`
header. Override either:

```go
app.NotFound(func(w http.ResponseWriter, r *http.Request) {
    _ = surf.JSONError(w, http.StatusNotFound, "no such endpoint")
})

app.MethodNotAllowed(func(w http.ResponseWriter, r *http.Request) {
    _ = surf.JSON(w, http.StatusMethodNotAllowed, map[string]string{
        "error":   "method not allowed",
        "allowed": w.Header().Get("Allow"),
    })
})
```

## Recovery from Panics

The Recovery middleware catches panics, logs the stack, and writes a generic
500 (or your `RecoveryHandler`):

```go
app.Use(surf.RecoveryWithDefaults())
```

```go
app.Use(surf.Recovery(surf.RecoveryConfig{
    Logger:    slog.Default(),
    StackSize: 4 << 10,
    RecoveryHandler: func(w http.ResponseWriter, r *http.Request, err interface{}) {
        _ = surf.JSONError(w, http.StatusInternalServerError, "internal error")
    },
}))
```

Always register Recovery first — it must wrap every other middleware to catch
their panics.
