# Routing

Surf uses a radix tree-based router that provides O(log n) route matching, making it extremely fast even with hundreds of routes.

## Basic Routes

Register routes for different HTTP methods:

```go
app := surf.NewApp()

app.Get("/users", listUsers)
app.Post("/users", createUser)
app.Put("/users/:id", updateUser)
app.Delete("/users/:id", deleteUser)
app.Patch("/users/:id", patchUser)
app.Head("/users", headUsers)
app.Options("/users", optionsUsers)
```

## Path Parameters

Use `:name` syntax to capture dynamic path segments:

```go
app.Get("/users/:id", func(w http.ResponseWriter, r *http.Request) error {
    id := surf.Param(r, "id")
    // id = "123" for request to /users/123
    return nil
})

app.Get("/posts/:postId/comments/:commentId", func(w http.ResponseWriter, r *http.Request) error {
    postId := surf.Param(r, "postId")
    commentId := surf.Param(r, "commentId")
    return nil
})
```

## Wildcard Routes

Use `*` to match any path suffix:

```go
app.Get("/files/*", func(w http.ResponseWriter, r *http.Request) error {
    path := surf.Param(r, "*")
    // path = "docs/readme.md" for request to /files/docs/readme.md
    return nil
})
```

<Callout type="info">
  Wildcard routes are useful for serving static files or implementing catch-all handlers.
</Callout>

## Route Groups

Organize routes with common prefixes and middleware:

```go
// Create a group with prefix
api := app.Group("/api/v1")

// Add group-specific middleware
api.Before(func(w http.ResponseWriter, r *http.Request) error {
    w.Header().Set("Content-Type", "application/json")
    return nil
})

// Register routes in the group
api.Get("/users", listUsers)      // GET /api/v1/users
api.Post("/users", createUser)    // POST /api/v1/users
api.Get("/users/:id", getUser)    // GET /api/v1/users/:id
```

### Nested Groups

Groups can be nested for better organization:

```go
api := app.Group("/api")

v1 := api.Group("/v1")
v1.Get("/users", listUsersV1)

v2 := api.Group("/v2")
v2.Get("/users", listUsersV2)  // Different implementation
```

### Group Middleware

Apply middleware to specific groups:

```go
// Public routes
public := app.Group("/public")
public.Get("/health", healthCheck)

// Protected routes with auth middleware
protected := app.Group("/admin")
protected.Before(authMiddleware)
protected.Get("/dashboard", dashboard)
protected.Get("/settings", settings)
```

## Handler Signature

All handlers use the same signature:

```go
type HandlerFunc func(w http.ResponseWriter, r *http.Request) error
```

Return `nil` for success or an `error` for failure:

```go
app.Get("/data", func(w http.ResponseWriter, r *http.Request) error {
    data, err := fetchData()
    if err != nil {
        return err  // Will be logged and return 500
    }

    return json.NewEncoder(w).Encode(data)
})
```

## Performance

The radix tree router provides significant performance benefits:

| Routes | Linear Search | Radix Tree | Improvement |
|--------|--------------|------------|-------------|
| 10     | ~500ns       | ~70ns      | 7x faster   |
| 100    | ~5,400ns     | ~74ns      | 73x faster  |
| 1000   | ~54,000ns    | ~78ns      | 692x faster |

<Callout type="tip">
  Route order doesn't matter with radix tree routing - performance is consistent regardless of registration order.
</Callout>
