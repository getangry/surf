# Static Files

Surf provides built-in support for serving static files with security features like directory traversal protection.

## Serving a Directory

Serve all files from a directory:

```go
app := surf.NewApp()

// Serve files from ./public at /static/*
app.Static("/static", "./public")

// Files are accessible at:
// /static/css/style.css  → ./public/css/style.css
// /static/js/app.js      → ./public/js/app.js
// /static/images/logo.png → ./public/images/logo.png
```

### Directory Structure Example

```
project/
├── main.go
└── public/
    ├── css/
    │   └── style.css
    ├── js/
    │   └── app.js
    └── images/
        └── logo.png
```

```go
app.Static("/assets", "./public")
```

## Serving a Single File

Serve individual files at specific paths:

```go
// Favicon
app.StaticFile("/favicon.ico", "./public/favicon.ico")

// Robots.txt
app.StaticFile("/robots.txt", "./public/robots.txt")

// Sitemap
app.StaticFile("/sitemap.xml", "./public/sitemap.xml")
```

## Security Features

### Directory Traversal Protection

Surf automatically blocks directory traversal attacks:

```go
app.Static("/files", "./uploads")

// These requests are blocked and return 404:
// /files/../../../etc/passwd
// /files/..%2F..%2F..%2Fetc%2Fpasswd
```

<Callout type="warn">
  Always use `app.Static()` instead of implementing your own file server to ensure proper security.
</Callout>

### File Existence Check

Non-existent files return 404 Not Found:

```go
// Request: GET /static/nonexistent.txt
// Response: 404 Not Found
```

### Directory Listing Disabled

Directory browsing is disabled by default:

```go
// Request: GET /static/
// Response: 404 Not Found (not a directory listing)
```

## Single Page Application (SPA) Support

`App.SPA` mounts a single-page application from any `fs.FS` — typically an
`embed.FS` baked into the binary at build time. It handles client-side routing
fallback (unknown paths serve the index document), long-cache headers for
fingerprinted assets, and a 404 list for paths that must not fall back to
HTML.

### Minimal SPA mount

```go
import (
    "embed"
    "io/fs"

    "github.com/getangry/surf"
)

//go:embed all:dist
var distFS embed.FS

func main() {
    app := surf.NewApp()

    sub, _ := fs.Sub(distFS, "dist")
    app.SPA("/", sub)
    // - GET / → index.html
    // - GET /assets/app.js → served with `Cache-Control: public, max-age=31536000, immutable`
    // - GET /any/client/route → falls back to index.html (200)

    app.Serve()
}
```

### With API routes alongside

Register the API first; the SPA mount catches everything else. Use
`ExcludePrefixes` to 404 unknown API paths instead of serving HTML:

```go
app := surf.NewApp()

// API routes
api := app.Group("/api")
api.Get("/users", listUsers)
api.Get("/posts", listPosts)

// SPA, with /api/* never falling back to index.html
app.SPAWithConfig(surf.SPAConfig{
    Prefix:          "/",
    FS:              sub,
    ExcludePrefixes: []string{"api"},
})
```

### Full configuration

```go
type SPAConfig struct {
    Prefix            string   // URL mount, e.g. "/" or "/app"
    FS                fs.FS    // filesystem to serve
    Index             string   // index document (default "index.html")
    ImmutablePrefixes []string // path segments served with immutable cache (default ["assets"])
    ExcludePrefixes   []string // path segments that 404 instead of falling back
}
```

The handler blocks directory traversal (`path.Clean` + `fs.FS.Open`'s
`fs.ValidPath` contract), so requests like `/../../etc/passwd` cannot escape
the served sub-tree.

## Complete Example

```go
package main

import (
    "embed"
    "io/fs"

    "github.com/getangry/surf"
)

//go:embed all:dist
var distFS embed.FS

func main() {
    app := surf.NewApp()

    app.Use(surf.RecoveryWithDefaults())

    // API routes
    api := app.Group("/api")
    api.Get("/users", listUsers)
    api.Get("/posts", listPosts)

    // Serve specific static files
    app.StaticFile("/robots.txt", "./public/robots.txt")

    // Single-page app — index fallback for client routing,
    // immutable cache on /assets/*, /api/* returns 404 instead of HTML.
    sub, _ := fs.Sub(distFS, "dist")
    app.SPAWithConfig(surf.SPAConfig{
        Prefix:          "/",
        FS:              sub,
        ExcludePrefixes: []string{"api"},
    })

    app.Serve()
}
```

## Content Types

Surf automatically sets the correct `Content-Type` header based on file extension:

| Extension | Content-Type |
|-----------|-------------|
| `.html` | `text/html` |
| `.css` | `text/css` |
| `.js` | `application/javascript` |
| `.json` | `application/json` |
| `.png` | `image/png` |
| `.jpg`, `.jpeg` | `image/jpeg` |
| `.svg` | `image/svg+xml` |
| `.woff2` | `font/woff2` |
